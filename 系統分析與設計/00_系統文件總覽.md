## 系統間整合關係

### 核心整合流程

#### 1. 採購到付款 (Procure to Pay)
```
PRM (請購/採購) → IVM (進貨入庫) → APM (應付帳款) → GLM (會計傳票)
```

#### 2. 訂單到收款 (Order to Cash)
```
OAM (銷售訂單) → IVM (出貨) → ARM (應收帳款) → GLM (會計傳票)
```

#### 3. 生產製造流程
```
OAM (訂單) → BOM (BOM 展開) → WIP (工單生產) → IVM (入庫)
```

#### 4. 人力成本流程
```
HRM (薪資計算) → GLM (人事費用傳票) → 成本分攤
```

## 資料庫設計原則

### 1. 命名規範
- **資料表**: `{模組}_{功能}` (如: hrm_employee)
- **語言檔**: `{表名}_l` (如: hrm_employee_l)
- **視圖**: `{表名}_v` (如: hrm_employee_v)
- **工作視圖**: `{表名}_w` (如: hrm_employee_w)

### 2. 標準欄位
所有資料表都包含:
- `entry_id`: 建立者
- `entry_date`: 建立日期
- `tr_id`: 異動者
- `tr_date`: 異動日期

### 3. 多語系支援
- 主檔 + 語言檔 (_l) + 視圖 (_v) 三層架構
- language_id 關聯語言別

### 4. 序列命名
- `{表名}_id` (如: hrm_employee_id)
